概述

项目结构

小程序的项目结构

  • 目录pages：存放所有小程序的页面
  • 目录utils：用来存放工具性质的模块（如格式化时间的自定义模块）
  • app.js ：小程序项目的入口文件
  • app.json：小程序项目的全局配置文件
  • app.wxss：全局样式文件
  • project.config.json：配置文件
  • sitemap.json：配置小程序页面是否允许被微信索引。

存放在pages目录下的每个页面，都由四个基本文件组成：

  • .js文件 —— 存放页面的数据，事件处理函数等
  • .json文件 —— 配置文件，配置窗口外观等
  • .wxml文件 —— 结构文件
  • .wxss文件 —— 样式表文件

app.json

app.json 是当前小程序的全局配置，包括了小程序的所有页面路径、窗口外观、界面表现、底部 tab 等。

  • pages：用来记录当前小程序所有页面的路径，首页为排第一位的页面。
  • window：全局定义小程序所有页面的背景色、文字颜色等
  • style：全局定义小程序组件所使用的样式版本
  • sitemapLocation：用来指明 sitemap.json 的位置
  • tabbar：定义底部tab，设置页面路径，名字，选中与未选中样式图表

project.config.json

project.config.json 是项目配置文件，用来记录我们对小程序开发工具所做的个性化配置，例如：

  • setting 中保存了编译相关的配置
  • projectname 中保存的是项目名称
  • appid 中保存的是小程序的账号 ID

sitemap.json

sitemap.json 文件用来配置小程序页面是否允许 微信索引。

当开发者允许微信索引时，微信会通过爬虫的形式，为小程序的页面内容建立索引。当用户的搜索关键字和页 面的索引匹配成功的时候，小程序的页面将可能展示在搜索结果中。

注意：sitemap 的索引提示是默认开启的，如需要关闭 sitemap 的索引提示，可在小程序项目配置文件 project.config.json 的 setting 中配置字段 checkSiteMap 为 false

WXML模板

WXML（WeiXin Markup Language）是小程序框架设计的一套标签语言，类似于HTML。

区别：

  • 标签名称不同
    • HTML （div, span, img, a）
    • WXML（view, text, image, navigator）
  • 属性节点不同
    • <a href=”#”>超链接</a>
    • <navigator url=”#”></navigator>
  • 提供了类似于 Vue 中的模板语法
    • 数据绑定
    • 列表渲染
    • 条件渲染

WXSS样式

是一套样式语言，与CSS的区别：

  • 新增rpx尺寸单位： rpx能在不同大小屏幕上的小程序自动进行换算。
  • 提供全局的样式和局部样式
    • 项目根目录的app.wxss会作用于所有小程序页面。
    • 局部页面的 .wxss样式仅对当前页面生效。
  • WXSS仅支持部分CSS选择器
    • .class 和 #id
    • element
    • 并集选择器 选择器1，选择器2{css} 、后代选择器 选择器1 选择器2 {css}
    • ::after 和 ::before 等伪类选择器

宿主环境

宿主环境指的是程序运行所必须的依赖环境。微信是小程序的宿主程序，借助微信，小程序可以：微信扫码、微信支付、微信登录、地理定位、etc…

小程序宿主环境包含：通信模型，运行机制，组件，API

通信模型

小程序通信的主体是渲染层和逻辑层。

  • WXML模板和WXSS样式工作在渲染层；
  • JS脚本工作在逻辑层

渲染层与逻辑层之间，逻辑层与第三方服务器间的通信，都是由微信帮忙转发的。

运行机制

小程序启动过程：

  1. 把小程序的代码包下载到本地
  2. 解析 app.json 全局配置文件
  3. 执行 app.js 小程序入口文件，调用 App() 创建小程序实例
  4. 渲染小程序首页
  5. 小程序启动完成

页面渲染过程：

  1. 加载解析页面的 .json 配置文件
  2. 加载页面的 .wxml 模板和 .wxss 样式
  3. 执行页面的 .js 文件，调用 Page() 创建页面实例
  4. 页面渲染完成

  • 图片组件，默认宽300px，高240px

    image组件的mode属性可以指定图片的裁剪和缩放模式：

    mode值 说明
    scaleToFill （默认值）缩放模式，不保持纵横比缩放图片，使图片的宽高完全拉伸至填满 image 元素
    aspectFit 缩放模式，保持纵横比缩放图片，使图片的长边能完全显示出来。也就是说，可以完整地将图片显示出来。
    aspectFill 缩放模式，保持纵横比缩放图片，只保证图片的短边能完全显示出来。也就是说，图片通常只在水平或垂直方 向是完整的，另一个方向将会发生截取。
    widthFix 缩放模式，宽度不变，高度自动变化，保持原图宽高比不变
    heightFix 缩放模式，高度不变，宽度自动变化，保持原图宽高比不变

API

分为3类：

  • 事件监听api

    特点：以on开头，用来监听某些事件的触发。

  • 同步api：以Sync结尾的api，可以通过函数返回值直接获取，如果执行出错会抛出异常。

  • 异步api：类似 JQuery 中的 $.ajax(options) 函数，需要通过success，fail，complete接收调用

条件渲染

  1. wx:if

使用 wx:if=”“ 来判断是否需要渲染该代码块。

  1. 结合 <block> 使用 wx:if

    如果要一次性控制多个组件的展示/隐藏，可以用 <block> 包起来。

    <block> 不是组件，只起包裹作用，不会被渲染。

  2. hidden

    使用 hidden=”“ 也能控制元素的显示与隐藏

    不同之处：

    • wx:if 以动态创建和移除元素的方式，控制元素的展示与隐藏 。
    • hidden 以切换样式的方式（display: none/block;）控制元素的显示与隐藏。

    建议：

    • 频繁切换时，建议使用 hidden
    • 控制条件复杂时，建议使用 wx:if 搭配 wx:elif、wx:else 进行展示与隐藏的切换

  3. wx:for

    指定数组，然后循环渲染重复的组件结构

    默认情况下，当前的索引用index表示，当前循环项用item表示。

    也可以指定index和item：

    wx:key —— 实现列表渲染时，建议为列表项指定唯一的key值，能提高渲染效率。

全局配置

app.json 是小程序的全局配置文件，常见配置如下：

  • pages：用来记录当前小程序所有页面的路径，首页为排第一位的页面。
  • window：全局定义小程序所有页面的背景色、文字颜色等。
  • style：是否启用新版的组件样式。
  • sitemapLocation：用来指明 sitemap.json 的位置。
  • tabbar：定义顶部或底部的tabBar，用于多页面的快速切换，最少设置2个，最多5个，设置页面路径，名字，选中与未选中样式图表。

window节点常用配置项：

属性名 类型 默认值 说明
navigationBarTitleText String 字符串 导航栏标题文字内容
navigationBarBackgroundColor HexColor #000000 导航栏背景颜色
navigationBarTextStyle String white 导航栏标题颜色，仅可用blcak和white
backgroundColor HexColor #fffff 窗口的背景色
backgroundTextStyle String dark 下拉loading的样式，仅可用dark和light
enablePullDownRefresh Boolean false 是否全局开启下拉刷新
onReachBottomDistance Number 50 页面上拉触底事件触发时据页面底部距离，单位为px

网络数据请求

处于安全考虑，小程序对数据接口的请求有以下限制：

  1. 只能请求HTTPS类型的接口；

  2. 必须将接口域名添加到信任列表；

    需要登录到小程序后台管理界面， 在开发—开发设置—服务器域名—修改request合法域名（只支持https，不能用ip，需经过ICP备案）

GET请求 —— 调用 wx.request()

POST请求 —— 调用 wx.request()

在页面刚加载时就请求数据： onLoad事件

接口数据：在代码/硅谷音乐_server 目录下cmd进入输入： npm start
