git

概述

git 是一个分布式版本控制系统，

版本控制 是一种记录文件内容变化，以便将来查阅特定版本修订情况的系统。

版本控制其实最重要的是可以记录文件修改历史记录，从而让用户能够查看历史版本，方便版本切换。

分布式的版本控制系统出现之后，解决了集中式版本控制系统的缺陷：

  1. 服务器断网的情况下也可以进行开发（因为版本控制是在本地进行的）
  2. 每个客户端保存的也都是整个完整的项目 (包含历史记录，更加安全)

工作机制

工作区——git add——> 暂存区——git commit——> 本地库

工作区：写代码；

暂存区：临时存储；

本地库：历史版本；

代码托管中心

代码托管中心是基于网络服务器的远程代码仓库，一般称为 远程库。

局域网：GitLab

互联网：GitHub，Gitee码云

基础操作命令

查看本地库状态：git status

添加暂存区：git add 文件名 （git add . //将所有修改加入暂存区）

提交本地库：git commit -m “日志信息” 文件名

工作区的文件是git对象，不能看作是项目，要添加到暂存区，才能形成一个项目。

查看简易版本信息：git reflog (参看详细版本信息：git log)

版本穿梭：git reset –hard 版本号

分支

在版本控制中，要同时推进多个任务，对每个任务就可以单独创建一个分支。

使用分支意味着程序员可以把自己的工作从开发主线上分离开来，开发自己的分支时不会影响主线分支的运行。（分支底层是指针的引用）

分支操作：

创建分支：git branch 分支名

查看分支：git branch -v

切换分支：git checkout 分支名

将指定分支合并到当前分支上：git merge 分支名

指定分支要提交本地库，合并后，当前分支才会修改。

合并冲突：

当两个分支修改了同一个文件，并都提交到本地库，那么合并时，系统发现一个文件出现两个版本，不知道应该选择哪一个，就是发生了冲突。需要手动修改，选择。

添加远程仓库

git remote -v ：查看当前所有远程地址别名

git remote add 别名 远程地址 ：添加远程库地址和别名

推送本地分支到远程仓库 ：git push 别名 分支

拉取远程库到本地库 ： git pull 别名 分支 （自动提交到本地库）

克隆远程库到本地库：git clone 远程地址

克隆结果：初始化本地仓库，创建别名

团队内协作

远程仓库的所有人需要在仓库的设置-成员设置中邀请其他团队成员加入，其他成员才能将本地分支推送到远程库

跨团队协作

团队外的人要先用fork获取远程库，修改后在界面的pull requests发送拉取请求。

SSH免密登录

生成 .ssh密钥目录： ssh-keygen -t rsa -C 账号邮箱

在.ssh目录下会有私钥id_rsa，公钥id_rsa.pub，将公钥添加到github上即可。
