概述

项目结构

小程序的项目结构

  • 目录pages：存放所有小程序的页面
  • 目录utils：用来存放工具性质的模块（如格式化时间的自定义模块）
  • app.js ：小程序项目的入口文件
  • app.json：小程序项目的全局配置文件
  • app.wxss：全局样式文件
  • project.config.json：配置文件
  • sitemap.json：配置小程序页面是否允许被微信索引。

存放在pages目录下的每个页面，都由四个基本文件组成：

  • .js文件 —— 存放页面的数据，事件处理函数等
  • .json文件 —— 配置文件，配置窗口外观等
  • .wxml文件 —— 结构文件
  • .wxss文件 —— 样式表文件

app.json

app.json 是当前小程序的全局配置，包括了小程序的所有页面路径、窗口外观、界面表现、底部 tab 等。

  • pages：用来记录当前小程序所有页面的路径，首页为排第一位的页面。
  • window：全局定义小程序所有页面的背景色、文字颜色等
  • style：全局定义小程序组件所使用的样式版本
  • sitemapLocation：用来指明 sitemap.json 的位置
  • tabbar：定义底部tab，设置页面路径，名字，选中与未选中样式图表

project.config.json

project.config.json 是项目配置文件，用来记录我们对小程序开发工具所做的个性化配置，例如：

  • setting 中保存了编译相关的配置
  • projectname 中保存的是项目名称
  • appid 中保存的是小程序的账号 ID

sitemap.json

sitemap.json 文件用来配置小程序页面是否允许 微信索引。

当开发者允许微信索引时，微信会通过爬虫的形式，为小程序的页面内容建立索引。当用户的搜索关键字和页 面的索引匹配成功的时候，小程序的页面将可能展示在搜索结果中。

注意：sitemap 的索引提示是默认开启的，如需要关闭 sitemap 的索引提示，可在小程序项目配置文件 project.config.json 的 setting 中配置字段 checkSiteMap 为 false

WXML模板

WXML（WeiXin Markup Language）是小程序框架设计的一套标签语言，类似于HTML。

区别：

  • 标签名称不同
    • HTML （div, span, img, a）
    • WXML（view, text, image, navigator）
  • 属性节点不同
    • <a href=”#”>超链接</a>
    • <navigator url=”#”></navigator>
  • 提供了类似于 Vue 中的模板语法
    • 数据绑定
    • 列表渲染
    • 条件渲染

WXSS样式

是一套样式语言，与CSS的区别：

  • 新增rpx尺寸单位： rpx能在不同大小屏幕上的小程序自动进行换算。
  • 提供全局的样式和局部样式
    • 项目根目录的app.wxss会作用于所有小程序页面。
    • 局部页面的 .wxss样式仅对当前页面生效。
  • WXSS仅支持部分CSS选择器
    • .class 和 #id
    • element
    • 并集选择器 选择器1，选择器2{css} 、后代选择器 选择器1 选择器2 {css}
    • ::after 和 ::before 等伪类选择器

宿主环境

宿主环境指的是程序运行所必须的依赖环境。微信是小程序的宿主程序，借助微信，小程序可以：微信扫码、微信支付、微信登录、地理定位、etc…

小程序宿主环境包含：通信模型，运行机制，组件，API

通信模型

小程序通信的主体是渲染层和逻辑层。

  • WXML模板和WXSS样式工作在渲染层；
  • JS脚本工作在逻辑层

渲染层与逻辑层之间，逻辑层与第三方服务器间的通信，都是由微信帮忙转发的。

运行机制

小程序启动过程：

  1. 把小程序的代码包下载到本地
  2. 解析 app.json 全局配置文件
  3. 执行 app.js 小程序入口文件，调用 App() 创建小程序实例
  4. 渲染小程序首页
  5. 小程序启动完成

页面渲染过程：

  1. 加载解析页面的 .json 配置文件
  2. 加载页面的 .wxml 模板和 .wxss 样式
  3. 执行页面的 .js 文件，调用 Page() 创建页面实例
  4. 页面渲染完成

条件渲染

  1. wx:if

使用 wx:if=”“ 来判断是否需要渲染该代码块。

  1. 结合 <block> 使用 wx:if

    如果要一次性控制多个组件的展示/隐藏，可以用 <block> 包起来。

    <block> 不是组件，只起包裹作用，不会被渲染。

  2. hidden

    使用 hidden=”“ 也能控制元素的显示与隐藏

    不同之处：

    • wx:if 以动态创建和移除元素的方式，控制元素的展示与隐藏 。
    • hidden 以切换样式的方式（display: none/block;）控制元素的显示与隐藏。

    建议：

    • 频繁切换时，建议使用 hidden
    • 控制条件复杂时，建议使用 wx:if 搭配 wx:elif、wx:else 进行展示与隐藏的切换

页面导航

页面导航： 页面间的相互跳转。小程序实现页面导航有两种方式：

  1. 声明式导航： 通过<navigator>导航组件实现跳转。
  2. 编程式导航：调用导航API。

声明式导航

  • 导航到 tabBar页面：

    <navigator>需要指定url和open-type属性。其中 url 以 / 开头， oprn-type 只能指定switchTab为跳转方式。

  • 导航到 非tabBar页面：

    <navigator>需要指定url和open-type属性。open-type 只能指定navigate为跳转方式。（open-type属性可省略）

  • 后退导航

    <navigator>需要指定open-type和delta属性。 其中open-type必须为 navigateBack，delta的值为后退的层级。（如果只退1级，delta可省略）

编程式导航

  • 导航到 tabBar页面

    调用 wx.switchTab(Object o) 方法，可跳转到 tabBar页面。参数对象的属性如下：

    属性 类型 时候必选 说明
    url string 是 需要跳转的tabBar页面路径，路径不能带参数
    success function 否 接口调用成功的回调函数
    fail function 否 失败时的回调函数
    compiete function 否 结束时的回调函数（一定会执行）

  • 导航到非tabBar页面

    调用 wx.navigateTo(Object o) ，参数对象的属性与 wx.switchTab() 一样，不过url属性能携带参数。

  • 后退导航

    调用 wx.navigateBack(Object o)方法。

导航传参

  • navigator组件的 url 的路径后可以携带参数。
  • 调用 wx.navigateTo(Object o) 时，url属性也能携带参数。

样式隔离

组件的样式隔离：

  • 默认情况下，不同组件间、组件与调用者间的样式不会相互影响；

  • app.wxss中的全局样式对组件无效；

  • 只有class选择器会有样式隔离效果， 而 id选择器、属性选择器、标签选择器不受样式隔离的影响。

    建议在组件和引用组件的页面中使用class选择器，不要用其他选择器，以避免干扰。

  • 修改组件的隔离性：在组件的 .js文件或 .jsonwen文件中通过 styleIsolation修改组件的样式隔离选项。

    styleIsolation的可选值：

    可选值 默认值 描述
    isolated 是 启动隔离样式，组件内外，使用class指定的样式不会相互影响
    apply-shared 否 页面的样式将影响自定义组件，而组件不会影响页面
    shared 否 会相互影响，也会影响其他设置了apply-shared或shared的组件

纯数据字段

纯数据字段：指那些不用于界面渲染的data字段。（即不展现在界面上，也不传递给其他组件，仅仅在组件内部使用）

组件的生命周期

生命周期函数 参数 描述说明
created 无 在组件实例刚刚被创建时执行
attached 无 组件实例进入页面节点时执行
ready 无 组件在视图层布局完成后执行
moved 无 组件实例被移动到节点树另一个位置时执行
datached 无 在组件实例从页面节点数移除时执行
error Object Error 每当组件方法抛出错误时执行

重要的三个生命函数：

  • 组件实例刚被创建好的时候，created 生命周期函数会被触发

    • 此时还不能调用 setData

    • 通常在这个生命周期函数中，只应该用于给组件的 this 添加一些自定义的属性字段

  • 在组件完全初始化完毕、进入页面节点树后， attached 生命周期函数会被触发

    • 此时， this.data 已被初始化完毕
    • 这个生命周期很有用，绝大多数初始化的工作可以在这个时机进行（例如发请求获取初始数据）

  • 在组件离开页面节点树后， detached 生命周期函数会被触发

    • 退出一个页面时，会触发页面内每个自定义组件的 detached 生命周期函数
    • 此时适合做一些清理性质的工作

定义： 在 lifetimes 内定义生命周期函数（推荐，lifetimes优先级最高）

    • 子组件的js中，通过 this.triggerEvent(‘事件名’, {参数对象})，将数据发给父组件；

    • 父组件的js中，在那个函数中通过e.detail获取数据；

  3. 获取组件实例

    在父组件里调用 this.selectComponent(“id/class选择器”)，获取子组件的实例对象，从而直接访问子组件的任何数据和方法。 调用时需要传入一个选择器。

behaviors

behaviors 用于实现组件间代码共享。

每个 behavior 可以包含一组属性、数据、生命周期函数和方法。组件引用它时，它的属性、数据和方法会被合并到组件中。

每个组件可以引用多个 behavior，behavior 也可以引用其它 behavior。

使用：

  • 创建behavior

  • 导入并使用behavior

    在组件中，使用require()方法导入需要的behavior，挂在后就可以访问了。

behaviors中所有可用的节点：

可用的节点 类型 描述
properties Object Map 属性
data Object 数据
methods Object 方法
behaviors String Array 引入其他的behavior
created Function 生命周期函数
attached Function 生命周期函数
ready Function 生命周期函数
moved Function 生命周期函数
detached Function 生命周期函数

同名字段的覆盖规则：

  • 使用者的字段 > 当前behavior > 调用的 behavior

分包

普通分包

分包指把一个完整的小程序项目，按需求划分为不同的子包，使用时按需加载。

好处：优化首次加载时间，多团队开发时更好解耦。

分包后的项目构成：1个主包 + 多个分包。

  • 主包：一般只包含启动页面和TabBar页面，以及所有分包都要的公共资源；
  • 分包：只包含和当前分包有关的页面和私有资源；

分包大小的限制： 主包+分包不超过16M， 单个包不超过2M

使用分包：在app.json中的 subpackages节点中声明分包的结构。

引用原则：

  1. 主包无法引用分包内的私有资源 ；
  2. 分包之间不能相互引用私有资源 ；
  3. 分包可以引用主包内的公共资源 ；

独立分包

独立分包 能独立于主包和其他分包而单独运行。 可以在不下载主包的情况下，独立运行。

因此，独立分包不能引用其他包的资源，包括主包的公共资源。

使用场景：让小程序从某一分包页面启动时，不用先下载主包。

分包预下载

分包预下载：在进入某页面时，自动下载之后可能需要的分包，从而提升进入其他分包页面的启动速度。

使用：

预下载的行为会在进入指定页面时触发。在app.json中，使用preloadRule节点定义分包的规则。

限制： 一个分包的预下载份额不超过2M，即预下载的分包加起来不能超过2M；
