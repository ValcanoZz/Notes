网页制作问题

VUE

  1. get请求的参数中带有英文点号，导致的请求错误：

    在Spring MVC框架中，会将地址中的最后一个点号截断为后缀名，从而判断其请求消费的内容类型，在不改变此策略的情况下，修改请求方式与地址也能达到正确处理的目标。

    如果请求的url中带有含有 英文点号 的参数，如地址、邮箱等，可以在请求方式中最后再加一个点号，或者修改后端的MVC框架策略，或者将 @PathVariable改为@RequestParam

  2. vue中设置 <form @submit.prevent> ，取消表单提交后的默认跳转

Nginx

（1）

使用 make upgrade 平滑升级时，最后发生错误：

原因可能是：之前启动nginx主进程实例没有用绝对路径的nginx命令,导致现在给这个进程发送信号没有效果：也就是没有使用 /usr/local/nginx/sbin/nginx 启动，而是通过查找环境变量中那个 nginx 命令启动的， 当平滑升级Nginx 在接收到 USR2 信号时使用 execve() 执行原 Nginx 启动命令行的第一个参数启动新实例时就会报找不到 nginx 可执行文件的错误（execve() 并不会查找 PATH 环境变量）这样子的话没办法进行平滑升级

解决：先杀死当前 nginx主进程，以绝对路径启动nginx，再执行make upgrade后不报错。
